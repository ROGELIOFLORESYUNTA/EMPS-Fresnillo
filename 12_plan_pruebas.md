# Plan de pruebas

## Caso 1: Proyecto simple

Sistema CRUD interno con 3 módulos, sin datos sensibles y requerimientos claros.

Esperado:
- Bytecoding reduce prototipo.
- Riesgo bajo.
- Mantenimiento básico.

## Caso 2: Proyecto municipal medio

Sistema con 8 módulos, reportes, roles, datos personales y capacitación.

Esperado:
- Modo híbrido recomendado.
- Revisión y pruebas relevantes.
- Mantenimiento mensual necesario.

## Caso 3: Proyecto con integración compleja

Sistema con integración externa, migración de datos y reglas críticas.

Esperado:
- No aplicar descuento fuerte por bytecoding.
- Riesgo alto.
- Requiere pruebas y arquitectura.

## Caso 4: Proveedor con cobro al final

Proyecto de 3 meses sin anticipo.

Esperado:
- Capital de trabajo alto.
- Alerta financiera.

## Caso 5: Cambio de alcance

Agregar módulo nuevo después de estimación.

Esperado:
- Se registra como nuevo alcance.
- Aumenta tiempo, costo y pruebas.
