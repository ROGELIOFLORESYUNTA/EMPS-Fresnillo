# Backlog de historias de usuario

## HU-01 Crear proyecto

Como estimador quiero crear un proyecto para registrar contexto, cliente y objetivo.

Criterios:
- Guarda nombre, área, objetivo y estado.
- Permite editar.

## HU-02 Capturar módulo

Como analista quiero capturar módulos para descomponer el alcance.

Criterios:
- Cada módulo tiene complejidad, claridad y criticidad.

## HU-03 Capturar historia

Como área usuaria quiero registrar lo que necesito para que el proveedor entienda el valor.

Criterios:
- Historia contiene actor, necesidad, beneficio y criterios.

## HU-04 Calcular escenarios

Como responsable técnico quiero comparar tres modos de desarrollo.

Criterios:
- Muestra tradicional, asistido y bytecoding.

## HU-05 Fiscal-laboral

Como proveedor quiero calcular costo real para no cotizar por debajo de mi costo.

Criterios:
- Muestra costo mensual por perfil y total.

## HU-06 Flujo de efectivo

Como proveedor quiero saber si puedo pagar antes de cobrar.

Criterios:
- Muestra capital de trabajo requerido.

## HU-07 Cambios

Como Ayuntamiento quiero registrar cambios para evitar confusión con garantía.

Criterios:
- El cambio se clasifica y estima.

## HU-08 Reportes

Como usuario quiero exportar reporte PDF para presentar la estimación.

Criterios:
- PDF se genera con resumen, costos, riesgos y recomendaciones.
